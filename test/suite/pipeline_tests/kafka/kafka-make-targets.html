üöÄ Basic Operations

‚ñ∂Ô∏è kafka-start
Starts the complete Kafka stack with all services and automatic topic creation.
make kafka-start

What it starts:

Zookeeper Coordination service on port 2181
Kafka Broker Message broker on port 9092
Kafka UI Web interface on port 8080
Setup Container Creates 3 sample topics automatically

Auto-created topics:

  • snaplogic-events (3 partitions)
  • snaplogic-logs (2 partitions)
  • snaplogic-metrics (1 partition)
‚èπÔ∏è kafka-stop
Stops all Kafka services and removes containers. Data volumes are preserved.
make kafka-stop

What it does:

‚úì Stops Kafka broker, Zookeeper, and UI
‚úì Removes containers (but keeps volumes)
‚úì Topics and messages are preserved
üîÑ kafka-restart
Performs a full restart of all Kafka services. Useful for applying configuration changes.
make kafka-restart

Sequence:

1. Stops all services
2. Waits 5 seconds
3. Starts all services again

üìã Topic Management

‚ûï kafka-create-topic
Creates a new Kafka topic with specified name and partition count.
make kafka-create-topic TOPIC=my-events PARTITIONS=5

Parameters:

TOPIC (Required) Name of the topic to create
PARTITIONS (Optional) Number of partitions (default: 1)

Examples:

make kafka-create-topic TOPIC=user-events
make kafka-create-topic TOPIC=logs PARTITIONS=10

üîß Troubleshooting Guide

üí°
Tip: Always run make kafka-status first to diagnose issues.

Port Already in Use

Error: "bind: address already in use"

Solution: Check if another service is using the ports:

lsof -i :9092 # Check Kafka port
lsof -i :8080 # Check UI port

Or modify ports in your .env file:

KAFKA_PORT=9093
KAFKA_UI_PORT=8085

Kafka Not Starting

Symptoms: Services appear running but broker doesn't respond

Solution:

make kafka-clean # Complete cleanup
make kafka-start # Fresh start

Cannot Create Topics

Error: "Error while executing topic command"

Possible causes:

  • Kafka broker not fully started (wait 30 seconds)
  • Zookeeper connection issues
  • Topic already exists

Solution:

make kafka-status # Check service health
make kafka-list-topics # Check existing topics

‚ú® Best Practices

üìö
Recommendations for optimal Kafka usage:

1. Topic Naming Convention

  • Use descriptive names: user-registration-events
  • Include environment: dev-payment-transactions
  • Use hyphens, not underscores
  • Keep names lowercase

2. Partition Strategy

  • Start with fewer partitions (1-3) for development
  • Increase partitions for high-throughput topics
  • Consider key-based partitioning for ordering
  • Monitor partition lag in Kafka UI

3. Development vs Production

  • Dev: Use kafka-dev-start for clean environment
  • Testing: Use kafka-start for pre-configured setup
  • CI/CD: Always run kafka-clean before tests
  • Production: Never use single-broker setup

4. Monitoring

  • Regularly check make kafka-status
  • Monitor consumer lag in Kafka UI
  • Use JMX metrics for detailed monitoring
  • Set up alerts for broker availability
