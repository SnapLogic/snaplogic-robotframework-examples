üîÑ Environment Loading Order

  1. Root .env loads first (base configuration)
  2. env_files/*.env load next (can override root settings)
  3. Command-line variables override everything

Example: make oracle-start ORACLE_PORT=1522

Docker Compose profiles control which services are accessible.

üì¶ Service Management

Starting Services

# Start all services with default profiles $ make start-services # Start specific service $ make oracle-start $ make kafka-start # Start with custom profiles $ make start-services COMPOSE_PROFILES=oracle-dev,kafka

Monitoring Services

# Check all services status $ make status # View specific service logs $ make oracle-logs $ make kafka-logs # Check network connectivity $ make network-check $ make container-networks

Stopping Services

# Stop all services $ make stop-services # Stop specific service $ make oracle-stop # Clean up everything (including volumes) $ make clean-all

üß™ Running Tests

# Run all tests $ make robot-run-tests # Run specific test suite $ make robot-run-tests SUITE=oracle_tests # Run tests in parallel $ make robot-run-tests PROCESSES=10 # Run with custom timeout $ make robot-run-tests TIMEOUT=60s

üîß Troubleshooting

Common Issues and Solutions

  • Port conflicts: Modify ports in env_files/.env.ports
  • Service won't start: Check logs with make [service]-logs
  • Network issues: Run make network-check
  • Permission errors: Ensure Docker daemon is running and user has permissions
  • Out of disk space: Run make clean-volumes to free up space

üìã Best Practices

  • ‚úÖ Always use make status before running tests
  • ‚úÖ Keep .env files organized and documented
  • ‚úÖ Use profiles to manage service groups
  • ‚úÖ Regularly clean up unused volumes and containers
  • ‚úÖ Monitor logs during service startup
  • ‚ùå Never commit sensitive credentials to version control
  • ‚ùå Don't modify docker-compose files directly - use .env for configuration
  • ‚ùå Avoid running services on conflicting ports
