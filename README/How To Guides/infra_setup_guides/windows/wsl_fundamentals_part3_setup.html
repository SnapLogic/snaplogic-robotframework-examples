The Execution Flow

1

Type make start-services in VS Code terminal

The command is entered in a terminal configured to use Ubuntu (WSL).

2

VS Code sends it to Ubuntu (WSL)

The WSL integration routes the command to the Linux environment.

3

Ubuntu's Bash reads the Makefile

The make tool parses the Makefile and resolves the target and its dependencies.

4

Makefile calls Docker Compose commands

The Makefile targets translate into docker-compose up commands for each service.

5

Docker starts containers (Oracle, Kafka, MinIO, etc.)

All services spin up on the snaplogicnet Docker bridge network.

6

Robot Framework tests execute against these containers

Tests create accounts, import pipelines, execute triggered tasks, and verify results.

The Dependency Chain

Each layer depends on the one below it. If any layer is missing or broken, everything above it fails.
Layer Stack
Layer 4: Robot Framework tests ← What you actually want to run Layer 3: Docker containers ← Where databases/services live Layer 2: WSL 2 + Ubuntu ← Where make/bash/docker commands run Layer 1: Windows + Docker Desktop ← The foundation

9. Step-by-Step Setup — Getting Everything Working Setup Guide

This section combines the fundamentals with practical setup steps to go from zero to a running environment.

1

Verify Windows Prerequisites

Open PowerShell (search "PowerShell" in Start menu):

PowerShell
# Check Windows version (need 10 v2004+ or Windows 11) winver # Check if virtualization is enabled # Task Manager → Performance → CPU → "Virtualization: Enabled"
2

Install WSL 2 with Ubuntu

Open PowerShell as Administrator (right-click → Run as Administrator):

PowerShell (Admin)
wsl --install
This single command: Enables the WSL feature, installs WSL 2, downloads and installs Ubuntu, and sets Ubuntu as the default. Restart your computer when prompted.

After restart, an Ubuntu window opens. Create your username and password (remember these!).

3

Verify the Installation

Back in PowerShell:

PowerShell
# Check WSL status wsl --status # Expected output: # Default Distribution: Ubuntu # Default Version: 2 # List all distros wsl --list --verbose # Expected: # * Ubuntu Running 2
4

Set Ubuntu as Default (if not already)

PowerShell
wsl --set-default Ubuntu
5

Install Development Tools Inside Ubuntu

Ubuntu (WSL)
# Enter Ubuntu wsl # Update package lists and install dev tools sudo apt update && sudo apt install -y build-essential git python3-pip # Verify make is installed make --version

6

Configure Docker Desktop for WSL

  1. Open Docker Desktop
  2. Click the gear icon → Settings
  3. Go to Resources → WSL Integration
  4. Check "Enable integration with my default WSL distro"
  5. Toggle ON next to Ubuntu
  6. Click "Apply & Restart"
7

Verify Docker Works Inside WSL

Ubuntu (WSL)
# Inside WSL/Ubuntu docker --version docker ps docker-compose --version

All three commands should return version info without errors.

8

Fix Docker Credentials for WSL (First-Time Only)

On a fresh Windows setup, Docker Desktop uses a credential helper that is not accessible from inside WSL. This causes an error getting credentials when pulling Docker images.

Ubuntu (WSL) — One-Time Fix
# Inside WSL/Ubuntu — create Docker config and fix credential helper mkdir -p ~/.docker echo '{"credsStore":"desktop.exe"}' > ~/.docker/config.json
Why this happens: Docker Desktop on Windows stores credentials using a Windows-native helper (wincred or desktop). When running docker commands from inside WSL, Docker looks for the credential helper inside Linux where it doesn't exist. Changing the config to desktop.exe tells Docker to use the Windows executable from WSL, which works because WSL can run .exe files from the Windows side.

If you still get the error after this fix:

Fallback Fix
# Clear the credential store entirely echo '{}' > ~/.docker/config.json
One-time fix: Once the config is correct, it persists across sessions. You only need to do this once.
9

Set Up VS Code

  1. Install the WSL extension in VS Code (Ctrl+Shift+X → search "WSL")
  2. Set Ubuntu as the default terminal: Ctrl+Shift+P → "Terminal: Select Default Profile" → Ubuntu (WSL)
  3. Or add to settings.json:
VS Code settings.json
{ "terminal.integrated.defaultProfile.windows": "Ubuntu (WSL)", "terminal.integrated.profiles.windows": { "Ubuntu (WSL)": { "path": "wsl.exe", "args": ["-d", "Ubuntu"], "icon": "terminal-ubuntu" } } }
10

Run the Project

VS Code Terminal (Ubuntu/WSL)
# Navigate to the project (note: C:\ becomes /mnt/c/) cd /mnt/c/Users/YourName/path/to/snaplogic-robotframework-examples # Enter the project folder (curly braces need double quotes) cd "{{cookiecutter.primary_pipeline_name}}" # Copy and configure environment cp .env.example .env # Edit .env with your SnapLogic credentials # Start services make start-services # Run tests make robot-run-all-tests TAGS="oracle" PROJECT_SPACE_SETUP=True
