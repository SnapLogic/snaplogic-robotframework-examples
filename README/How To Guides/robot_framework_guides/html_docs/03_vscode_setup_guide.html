1. Required Extensions

Install these extensions from the VS Code marketplace for the best Robot Framework development experience:

Extension Publisher Status Purpose
Robot Framework Language Server Robocorp Required Syntax highlighting, autocomplete, go-to-definition for .robot and .resource files
Python Microsoft Required Python support for custom libraries and debugging
Docker Microsoft Recommended View running containers, logs, and Docker Compose management
YAML Red Hat Recommended Syntax support for docker-compose.yml and .travis.yml
Makefile Tools Microsoft Optional Makefile syntax highlighting and target navigation
DotENV mikestead Optional Syntax highlighting for .env files

Quick Install via Command Line

# Install required extensions
code --install-extension robocorp.robotframework-lsp
code --install-extension ms-python.python

# Install recommended extensions
code --install-extension ms-azuretools.vscode-docker
code --install-extension redhat.vscode-yaml

# Install optional extensions
code --install-extension ms-vscode.makefile-tools
code --install-extension mikestead.dotenv

2. Workspace Configuration

Opening the Project

Open the generated project directory (not the root template repository) as your workspace:

# Open the generated project directory in VS Code
code /path/to/snaplogic-robotframework-examples/your_pipeline_name

# Or from within the directory
cd your_pipeline_name
code .
Important: Make sure your VS Code workspace root is the generated project folder (e.g., your_pipeline_name/), not the parent template directory. This ensures the Robot Framework Language Server can properly resolve resource imports and file paths.

Recommended settings.json

Add these settings to your workspace .vscode/settings.json:

{
    // Robot Framework Language Server
    "robot.language-server.python": "/usr/local/bin/python3",
    "robot.pythonpath": [
        "${workspaceFolder}/test/libraries",
        "${workspaceFolder}/test/resources"
    ],

    // Python
    "python.defaultInterpreterPath": "/usr/local/bin/python3",

    // File Associations
    "files.associations": {
        "*.robot": "robotframework",
        "*.resource": "robotframework",
        ".env*": "dotenv",
        "Makefile*": "makefile"
    },

    // Editor Settings
    "editor.tabSize": 4,
    "editor.insertSpaces": true,
    "editor.rulers": [120],

    // Search Exclusions (reduce noise)
    "search.exclude": {
        "**/robot_output/**": true,
        "**/__pycache__/**": true,
        "**/node_modules/**": true
    }
}

3. Python Environment

While tests run inside Docker, a local Python environment helps with IDE features like autocomplete and go-to-definition.

Setting Up a Local Virtual Environment

# Create a virtual environment
python3 -m venv .venv

# Activate it
source .venv/bin/activate   # macOS/Linux
.venv\Scripts\activate      # Windows

# Install Robot Framework and key dependencies
pip install robotframework
pip install robotframework-requests
pip install robotframework-jsonlibrary
pip install snaplogic-common-robot

# Select this interpreter in VS Code (Cmd+Shift+P -> Python: Select Interpreter)
Note: The local environment is only for IDE support. Actual test execution always happens inside the Docker tools container, which has all dependencies pre-installed including database drivers.

4. Code Navigation Tips

Go to Definition (F12)

  • Click on any keyword and press F12 to jump to its definition
  • Works for custom keywords in .resource files
  • Works for keywords from imported libraries
  • Works for variable definitions

Find All References (Shift+F12)

  • Find where a keyword is used across all files
  • Useful for understanding keyword dependencies
  • Works for variables and resources

Symbol Search (Cmd+T)

  • Search for keywords, variables, and test cases by name
  • Quick navigation across the project
  • Supports fuzzy matching

File Search (Cmd+P)

  • Quickly open any file by typing part of its name
  • Type *.robot to filter test files
  • Type *.resource to find resource files

6. Productivity Tips

Integrated Terminal

  • Use VS Code's built-in terminal (Ctrl+`)
  • Run make commands directly: make status
  • Split terminals for logs + commands side by side
  • Use Docker extension to view container logs visually

Viewing Test Reports

  • After tests run, open test/robot_output/report-*.html
  • Right-click the file and select "Open with Live Server"
  • Or open directly in your browser
  • Use log-*.html for detailed step-by-step execution
