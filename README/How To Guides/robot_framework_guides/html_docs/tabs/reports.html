Overview

When you run Robot Framework tests using any of the make commands, three essential files are automatically generated in the test/robot_output directory:

üìç Output Location: /snaplogic-robotframework-examples/test/robot_output/

This directory contains all test execution reports, logs, and raw output data from your most recent test run.

Generated Files

File Purpose Format Primary Use
report.html High-level test execution summary Interactive HTML Quick overview of test results
log.html Detailed step-by-step execution log Interactive HTML Debugging and detailed analysis
output.xml Raw test execution data XML CI/CD integration and reprocessing

Report.html - Test Execution Report

The report.html file is your primary test execution report that provides a comprehensive overview of your test run.

Key Features of report.html:

  • Summary Dashboard: Overall pass/fail status at a glance
  • Test Statistics: Breakdown by tags, suites, and individual tests
  • Execution Timeline: Start time, end time, and total duration
  • Pass/Fail Visualization: Color-coded bars showing test results
  • Direct Link to Log: Click to navigate to detailed logs
  • Tag Statistics: Results filtered by test tags
  • Suite Statistics: Performance metrics per test suite

Log.html - Detailed Execution Log

The log.html file contains the complete, step-by-step execution details of your test run.

What You'll Find in log.html:

  • Keyword Execution: Every keyword executed with timestamps
  • Variable Values: Actual values of variables during execution
  • API Responses: Full request/response details for API calls
  • Error Messages: Detailed error traces and stack traces
  • Screenshots: Embedded screenshots (if configured)
  • Expandable Details: Click to expand/collapse execution details

üí° Pro Tip: Use log.html when debugging failed tests. It shows exactly what happened at each step, including variable values and system responses.

Output.xml - Raw Test Data

The output.xml file contains all test execution data in a structured XML format.

Common Uses for output.xml:

  • CI/CD Integration: Parse results in Jenkins, GitLab CI, etc.
  • Report Regeneration: Recreate HTML reports with different settings
  • Custom Analysis: Process with custom scripts or tools
  • Test Metrics: Extract data for dashboards and KPIs
  • Historical Analysis: Compare results across multiple runs

Accessing Reports

Method 1: Direct File Access

# Navigate to the output directory
cd test/robot_output

# Open report in default browser (Mac)
open report.html

# Open report in default browser (Linux)
xdg-open report.html

# Open report in default browser (Windows)
start report.html

Method 2: Via File Explorer

  1. Navigate to snaplogic-robotframework-examples/test/robot_output/
  2. Double-click report.html to open in your default browser
  3. Click the "Log" link within the report to view detailed logs

Method 3: Automatic Opening (Configure in Makefile)

# Some make targets can be configured to auto-open reports
make robot-run-tests TAGS="oracle" OPEN_REPORT=true

Understanding Report Contents

Report Status Indicators

Status Color Meaning
PASS Green Test executed successfully
FAIL Red Test execution failed
SKIP Yellow Test was skipped

Navigation Tips

  • Expand/Collapse: Click on suite or test names to expand details
  • Filter by Status: Use checkboxes to show only passed/failed tests
  • Search: Use Ctrl+F to search for specific keywords or errors
  • Timeline View: Hover over execution bars to see timing details
  • Tag Filtering: Click on tags to filter tests by specific tags

Best Practices

1. Report Management

  • Archive Important Reports: Copy reports for significant test runs to a separate archive folder
  • Add Timestamps: Reports are overwritten by default, so rename important ones with timestamps
  • Version Control: Don't commit report files to Git (they're already in .gitignore)

2. Debugging with Reports

  • Start with report.html: Get the overview of what failed
  • Drill into log.html: Find the exact step where failure occurred
  • Check Variable Values: Look at variable states before failure
  • Review API Calls: Examine request/response payloads

3. Sharing Reports

  • Self-Contained: HTML reports are self-contained and can be shared via email
  • Screenshots: Reports include embedded data, no external dependencies
  • Compress for Sharing: Zip the entire robot_output folder to preserve all files

4. Custom Report Generation

# Generate report with custom title
robot --reporttitle "Nightly Regression Tests" tests/

# Generate report without log file
robot --log NONE tests/

# Generate report with custom name
robot --report custom_report.html tests/

# Combine multiple output files
rebot --name "Combined Tests" output1.xml output2.xml

‚úÖ Success Tip: After each test run, always check the report.html first. If tests failed, click through to log.html for detailed debugging information. The combination of these reports provides complete visibility into your test execution.

Reports are your window into test execution. Use them effectively to quickly identify issues, understand test behavior, and maintain high-quality test automation.
