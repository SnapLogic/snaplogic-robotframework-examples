2. Execution Flow

Here's the step-by-step sequence when tests start:

1
Robot Framework Starts
The robot command is executed (via Make target or directly). Robot Framework begins processing the test suite directory.
2
Process *** Settings *** Section
Robot Framework reads the Settings section of each file, importing libraries, resources, and identifying Suite Setup declarations.
3
Execute __init__.robot
The test/suite/__init__.robot file is automatically detected and executed ONCE before all test suites in the directory.
4
Run Suite Setup: "Before Suite"
The Before Suite keyword defined in __init__.robot executes, which triggers all initialization steps.
5
Load Environment Variables
All .env files are loaded from the env_files/ directory and root, with root having highest precedence.
6
Validate Environment Variables
Required variables (URL, ORG_ADMIN_USER, ORG_ADMIN_PASSWORD, etc.) are validated. Suite fails if any are missing.
7
Set Up Global Variables
Path variables and other configuration are made globally available to all test cases using Set Global Variable.
8
Project Set Up
Authenticates with SnapLogic API, optionally deletes/creates project space, and sets up required accounts based on PROJECT_SPACE_SETUP flag.
9
Test Cases Execute
Individual test cases run with access to all global variables and configured environment.

3. The __init__.robot File

The __init__.robot file is a special Robot Framework file that runs once before all tests in its directory.

File Location

test/suite/__init__.robot

Structure

*** Settings *** Library OperatingSystem Library BuiltIn Library Process Library JSONLibrary Resource snaplogic_common_robot/.../snaplogic_keywords.resource *** Variables *** ${env_file_path} ${CURDIR}/../../.env ${account_payload_path} ${CURDIR}/test_data/accounts_payload ${pipeline_payload_path} ${CURDIR}/test_data/pipelines *** Keywords *** Suite Setup Before Suite Before Suite Log To Console env_file_path is:${env_file_path} Load Environment Variables Validate Environment Variables Set Up Global Variables Project Set Up-Delete Project Space-Create New Project space-Create Accounts
Key Point: The __init__.robot file runs automatically when Robot Framework discovers it in a test directory. It's executed ONCE before any test suites in that directory.

4. Environment File Loading

Environment variables are loaded from multiple .env files with a specific precedence order.

Loading Order (Precedence)

Order Location Precedence Description
1 env_files/*.env* Lowest Files in env_files root directory
2 env_files/database_accounts/ Low Database-specific credentials (.env.snowflake, .env.postgres, etc.)
3 env_files/groundplex/ Medium Groundplex port configurations
4 env_files/messaging_service_accounts/ Medium Kafka, JMS configurations
5 env_files/mock_service_accounts/ Medium S3, Salesforce mock configurations
6 .env (root) Highest Root .env file - overrides all others
Important: Variables defined in the root .env file will OVERRIDE any variables with the same name from env_files/ subdirectories.

5. Setup Keywords

These keywords execute during the Before Suite setup:

5.1 Load Environment Variables

Load Environment Variables # Lists files in env_files/ root # Loads each .env* file found # Recursively processes subdirectories # Finally loads root .env (highest precedence) Log To Console Loading environment variables... @{env_files}= List Files In Directory ${env_files_dir} pattern=.env* FOR ${file} IN @{env_files} Load Single Env File ${env_files_dir}/${file} END # Load root .env last (highest precedence) Load Single Env File ${env_file_path}

5.2 Validate Environment Variables

Validate Environment Variables @{required_vars}= Create List ... URL ... ORG_ADMIN_USER ... ORG_ADMIN_PASSWORD ... ORG_NAME ... PROJECT_SPACE ... PROJECT_NAME ... GROUNDPLEX_NAME FOR ${var} IN @{required_vars} ${value}= Get Environment Variable ${var} default=NOT_SET Should Not Be Equal ${value} NOT_SET ... msg=Required environment variable ${var} is not set! END

5.3 Set Up Global Variables

Set Up Global Variables Set Global Variable ${account_payload_path} Set Global Variable ${env_file_path} Set Global Variable ${pipeline_payload_path} Set Global Variable ${generative_slp_pipelines_path}

5.4 Project Set Up

Project Set Up-Delete Project Space-Create New Project space-Create Accounts # Calls Set Up Data from snaplogic_keywords.resource Set Up Data ... url=${URL} ... org_admin_user=${ORG_ADMIN_USER} ... org_admin_password=${ORG_ADMIN_PASSWORD} ... org_name=${ORG_NAME} ... project_space=${PROJECT_SPACE} ... project_name=${PROJECT_NAME} ... account_payload_path=${account_payload_path} ... project_space_setup=${PROJECT_SPACE_SETUP}
What Set Up Data Does:
  • Authenticates with SnapLogic API
  • Retrieves organization snode ID
  • If PROJECT_SPACE_SETUP=True: Deletes existing project space and creates new one
  • Creates required accounts from payload files
  • Sets ORG_SNODE_ID as global variable

6. Variable Scopes

Understanding how variables are scoped in the test framework:

Variable Type Set By Accessible In Scope
OS Environment Variables Set Environment Variable Any subprocess, system calls Process-wide
Robot Scalar ${VAR} Set Global Variable All test cases & keywords Suite-wide
Robot Dictionary &{VAR} Auto-parsed from JSON All test cases & keywords Suite-wide
Robot List @{VAR} Auto-parsed from JSON All test cases & keywords Suite-wide

Variable Lifecycle

    .env Files                Robot Variables              Test Cases
    =========                 ===============              ==========

    KEY=value      -->    Set Environment Variable    -->   %{KEY}
                          Set Global Variable ${KEY}  -->   ${KEY}

    JSON_DICT={...} -->   &{json_dict}                -->   &{json_dict}[key]

    JSON_LIST=[...] -->   @{json_list}                -->   @{json_list}[0]
