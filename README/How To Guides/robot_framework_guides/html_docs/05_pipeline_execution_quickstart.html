1. The 5-Step Quick Start

1 Install Docker Desktop 
# Download from https://www.docker.com/products/docker-desktop
# Verify installation:
docker --version
docker compose version
2 Clone & Generate Project 
git clone <repository-url>
cd snaplogic-robotframework-examples
make generate-project
3 Navigate to Generated Project 
# Option A: Auto-navigate
eval $(make change-dir)

# Option B: Manual
cd your_pipeline_name
4 Configure Environment 
cp .env.example .env

# Edit .env with your SnapLogic credentials:
# - URL (SnapLogic instance URL)
# - ORG_ADMIN_USER (your admin email)
# - ORG_ADMIN_PASSWORD (your password)
# - ORG_NAME (your organization name)
vi .env
5 Start Services & Run Tests 
# Start Docker services
make start-services

# Run tests (first time - creates project space + Groundplex)
make robot-run-all-tests TAGS="oracle" PROJECT_SPACE_SETUP=True

# Subsequent runs (project space already exists)
make robot-run-all-tests TAGS="oracle"
That's it! Your tests are now running. Results will be available in test/robot_output/ as HTML reports.

2. Two Execution Paths

Path A: Framework-Managed Groundplex (Recommended)

The framework creates and manages the Groundplex for you.

# First time (full setup)
make start-services
make robot-run-all-tests \
  TAGS="oracle" \
  PROJECT_SPACE_SETUP=True

# Subsequent runs
make robot-run-all-tests TAGS="oracle"
  • Creates project space and Snaplex
  • Launches Groundplex container
  • Runs tests
  • Best for: CI/CD, first-time setup

Path B: Existing Groundplex

Use when you already have a running Groundplex.

# First time (setup project only)
make start-services
make robot-run-tests-no-gp \
  TAGS="oracle" \
  PROJECT_SPACE_SETUP=True

# Subsequent runs
make robot-run-tests-no-gp TAGS="oracle"
  • Skips Groundplex creation/launch
  • Faster execution
  • Assumes GP is already running
  • Best for: development, quick iteration

PROJECT_SPACE_SETUP Parameter

ValueBehaviorWhen to Use
True Deletes existing project space, creates new one, sets up Snaplex First-time setup, clean environment, CI/CD pipelines
False (default) Verifies project exists, fails if missing Subsequent runs, development, stable environments

4. Test Tags for Selective Execution

Tags control which tests run. Use them to execute only the tests you need:

# Run Oracle tests only
make robot-run-all-tests TAGS="oracle"

# Run PostgreSQL demo tests
make robot-run-all-tests TAGS="postgres_demo"

# Run Snowflake tests
make robot-run-all-tests TAGS="snowflake_demo"

# Run smoke tests (quick validation)
make robot-run-all-tests TAGS="smoke"

# Combine tags with OR logic (any tag matches)
make robot-run-all-tests TAGS="oracle OR postgres"

# Combine tags with AND logic (all tags must match)
make robot-run-all-tests TAGS="snowflake_demo AND task_creation"

# Exclude specific tests
make robot-run-all-tests TAGS="oracle AND NOT cleanup"

5. Viewing Test Results

After test execution, reports are generated in test/robot_output/:

FilePurposeBest For
report-*.html Summary report with pass/fail statistics Quick overview of results
log-*.html Detailed execution log with step-by-step output Debugging failures, understanding flow
output-*.xml Machine-readable XML output CI/CD integration, automated processing 
# Open the latest report in your browser (macOS)
open test/robot_output/report-*.html

# Or find the exact filename
ls -la test/robot_output/
