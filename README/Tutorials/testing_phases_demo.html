Production Deployment

Deploy validated pipelines to production

‚úÖ Quality Gates

  • All tests passed in QA
  • Performance benchmarks met
  • Data validation successful
  • No errors in pipeline execution
  • Code review approved
  • Security scan passed

üöÄ Production Steps

  • Deploy approved pipelines only
  • Update with production credentials
  • Configure production schedules
  • Run smoke tests (limited)
  • Monitor pipeline execution
  • No test data in production

üìù Key Points

  • Phase 2 (test data preparation) ensures all validation inputs are ready before environment setup
  • Phase 3 (.env configuration) MUST happen before Phase 4 (test creation)
  • Test data files are used by the framework APIs for pipeline validation
  • The .env file contains all credentials and configurations needed by the framework APIs
  • Framework APIs (snaplogic-common-robot) rely on environment variables from .env
  • Docker containers load both the .env file and test data files
  • This ensures consistent configuration and test data across all test environments
